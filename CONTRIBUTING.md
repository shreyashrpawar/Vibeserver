# Contributing to Vibeserver

Thank you for your interest in contributing to Vibeserver! We welcome contributions from the community to help make this project better.

## Getting Started
1. **Fork the repository** on GitHub.
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/shreyashrpawar/Vibeserver.git
   cd Vibeserver
   ```
3. **Install dependencies**:
   ```bash
   npm install
   ```
4. **Run the development server**:
   ```bash
   npm run dev
   ```

## How to Contribute
- **Reporting Bugs**: Open an issue describing the bug, how to reproduce it, and your environment.
- **Suggesting Enhancements**: Open an issue describing your idea and why it would be useful.
- **Pull Requests**:
    - Create a new branch for your feature or fix (`git checkout -b feature/amazing-feature`).
    - Commit your changes with descriptive messages.
    - Push to your fork (`git push origin feature/amazing-feature`).
    - Open a Pull Request from your branch to our `main` branch.

## Code Style
- We use **Prettier** and **ESLint** to maintain code quality.
- Ensure your code passes linting before submitting a PR:
  ```bash
  npm run lint
  ```
